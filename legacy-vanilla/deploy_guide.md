# GitHub Pages デプロイ手順ガイド

このアプリケーションは完全にサーバーレスの静的SPA（シングルページアプリケーション）ですので、**GitHub Pages** を使用して完全無料で全世界からアクセスできるように公開（デプロイ）できます。

以下に、PCとスマートフォンからアクセスするためのデプロイ手順を解説します。

---

## 🚀 デプロイ手順 (3ステップ)

### 1. GitHubリポジトリの作成とプッシュ
すでにGitHubアカウントをお持ちですので、新しくリポジトリを作成してファイルをアップロードします。

1. [GitHub](https://github.com/)にアクセスし、ログインします。
2. 新しいリポジトリを作成します（例: リポジトリ名 `Language_studying`）。
   - 設定：**Public (公開)** に設定します。（※GitHub Pagesの無料枠はPublicリポジトリである必要があります。APIキー等はGistに非公開保存され、ソースコード上には露出しないため安全です。）
3. このプロジェクトフォルダの全ファイルを、コマンドラインまたはGitHubのウェブ画面からリポジトリの `main` ブランチにプッシュ（アップロード）します。

   *コマンドラインでの実行例:*
   ```bash
   git init
   git add .
   git commit -m "Initial commit of Callan AI Tutor"
   git branch -M main
   git remote add origin https://github.com/あなたのユーザー名/Language_studying.git
   git push -u origin main
   ```

### 2. GitHub Pages の有効化
アップロードしたリポジトリをWebサイトとして公開します。

1. 作成したGitHubリポジトリのページを開き、上部タブの **「Settings (設定)」** をクリックします。
2. 左メニューの「Code and automation」カテゴリにある **「Pages」** をクリックします。
3. **「Build and deployment」** の項目で以下のように設定します：
   - **Source**: `Deploy from a branch` (デフォルト)
   - **Branch**: `main` (または公開したいブランチ名) を選び、フォルダを `/ (root)` に設定します。
4. **「Save」** ボタンをクリックします。

### 3. デプロイ完了とアクセス
1. 設定保存後、1〜2分待つと、同じ画面（Pagesの設定画面）の上部に以下のような公開サイトのURLが表示されます。
   `https://yusuke9357.github.io/Language_studying/`
2. このURLをPCやスマホのブラウザで開いて動作を確認してください！

## 📱 スマートフォンでの快適な使い方（ホーム画面に追加）

スマートフォンからアクセスした際、SafariやChromeのメニューバーを非表示にして、まるで**ネイティブアプリ**のような全画面モードで快適に学習を進めることができます。

### iOS (Safari) の場合
1. Safariで公開したURL（`https://yusuke9357.github.io/Language_studying/`）を開きます。
2. 下部ツールバーの **「共有（アクション）ボタン」**（四角から上矢印が出ているアイコン）をタップします。
3. メニューをスクロールし、**「ホーム画面に追加」** をタップします。
4. アプリ名を確認し、右上の「追加」をタップするとホーム画面にアイコンが表示されます。

### Android (Chrome) の場合
1. Chromeで公開したURLを開きます。
2. 右上のメニューアイコン（縦の3点リーダー）をタップします。
3. **「ホーム画面に追加」** または **「アプリをインストール」** をタップします。
4. 画面の指示に従って追加するとホーム画面にショートカットが追加されます。

## 🔒 セキュリティとデータ同期について
- デプロイした公開サイトにアクセスすると、初回のみ **ユーザーログイン** と **APIキー / GitHub Token / Gist ID** の入力を求められます。
- 入力したキーやトークンは、ブラウザを閉じても消えないローカル保存領域（`localStorage`）に保存され、**外部のサーバーに送信されることはありません**（Gemini APIおよびGitHub APIへのリクエストは、すべてあなたの端末のブラウザから直接Google/GitHubに向けて送信されます）。
- 異なるPCやスマートフォンからログインした場合も、同じ **GitHub Token** と **Gist ID** を設定することで、クラウド（Gist）経由で同じユーザー名の学習履歴や進捗が完全に同期されます。
